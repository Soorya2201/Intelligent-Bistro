================================================================================
  SECTION 0 — PROJECT OVERVIEW & GOAL
================================================================================

You are building "The Intelligent Bistro" — a mobile food ordering app where
users can order food through a conversational AI assistant. The AI understands
natural language, updates a live shopping cart in real-time while it is still
streaming a response, supports voice input and voice output, remembers dietary
preferences, proactively suggests pairings, and handles ambiguous orders with
interactive quick-reply chips.

This must be production-quality. Every screen must be polished and intentional.

--------------------------------------------------------------------------------
  KEY TECHNICAL INNOVATIONS (What makes this stand out)
--------------------------------------------------------------------------------

1. SENTINEL STREAMING: The AI embeds structured JSON inside its natural language
   response using sentinel delimiters. The frontend parses this mid-stream and
   updates the cart BEFORE the AI finishes speaking. This creates the illusion
   that the cart is alive and reacting to the AI's voice in real time.

2. JARVIS DOT CLUSTER: A custom CSS animation — a cluster of warm-colored dots
   that pulse, orbit, and breathe while the AI is speaking. No libraries.
   Pure CSS keyframes. This is the visual identity of the AI assistant.

3. VOICE-FIRST: Tap to talk, AI transcribes via native STT, responds in natural
   language, speaks the response back via TTS. The cart animates the whole time.

4. DIETARY MEMORY: The AI remembers "I'm vegan" across the session and
   auto-warns on conflicting items.

5. AMBIGUITY CHIPS: When the AI is unsure, it renders interactive quick-reply
   buttons (not just text) so the user taps to clarify.

================================================================================
  SECTION 6 — THE SENTINEL STREAM PARSER (CRITICAL)
================================================================================

This is the most important piece of the frontend. Study it carefully.

--------------------------------------------------------------------------------
  HOW SENTINELS WORK
--------------------------------------------------------------------------------

The AI backend instructs Claude to respond like this:

  "I've added that for you! ✦ACTION✦{"op":"add","items":[{"id":"truffle-fries",
  "name":"Truffle Fries","qty":1,"price":8.50}]}✦END✦ Would you like anything
  else?"

The frontend receives this as a stream of text chunks (Server-Sent Events).
As each chunk arrives, we accumulate it into a buffer.

The parser runs on every new chunk and:
  1. Looks for the opening sentinel: ✦ACTION✦
  2. Once found, begins buffering the JSON
  3. When it finds ✦END✦, extracts the JSON, parses it, and dispatches the action
  4. Strips the entire sentinel block from the visible chat text
  5. Only clean natural language appears in the chat bubble

--------------------------------------------------------------------------------
  FILE: apps/mobile/src/hooks/useStreamParser.ts
--------------------------------------------------------------------------------

SENTINEL CONSTANTS (define at the top of the file):
  const SENTINEL_OPEN = '✦ACTION✦';
  const SENTINEL_CLOSE = '✦END✦';

STATE (use useRef for these, not useState, to avoid re-render on each chunk):
  bufferRef: string          ← accumulates raw text from the stream
  insideActionRef: boolean   ← are we currently inside a sentinel block?
  actionBufferRef: string    ← accumulates JSON characters

FUNCTION: parseChunk(rawChunk: string): { visibleText: string, actions: CartAction[] }

  PSEUDOCODE:
    visibleText = ''
    actions = []

    for each character c in rawChunk:
      if not insideAction:
        append c to buffer
        if buffer ends with SENTINEL_OPEN:
          insideAction = true
          actionBuffer = ''
          // Remove the sentinel open from visibleText
          visibleText = visibleText.slice(0, visibleText.length - SENTINEL_OPEN.length + 1)
          // Actually: strip it from the end of accumulated visible output
        else:
          visibleText += c
      else:
        actionBuffer += c
        if actionBuffer ends with SENTINEL_CLOSE:
          jsonStr = actionBuffer.slice(0, -SENTINEL_CLOSE.length)
          try:
            action = JSON.parse(jsonStr)
            actions.push(action)
          catch:
            // malformed JSON — log but don't crash
          insideAction = false
          actionBuffer = ''
          // Do NOT add anything to visibleText — sentinel is invisible

    return { visibleText, actions }

IMPORTANT IMPLEMENTATION NOTE:
  The character-by-character approach above is the most reliable. An alternative
  is string indexOf() checking after each append, but char-by-char is cleaner.

HOOK RETURN VALUE:
  return {
    processChunk: (rawChunk) => { ... },    ← call this on each SSE chunk
    reset: () => { ... }                     ← call before each new message
  }

WHEN AN ACTION IS RECEIVED:
  Immediately call the appropriate store action:
  - op === 'add'     → store.addItem(...)
  - op === 'remove'  → store.removeItem(...)
  - op === 'update'  → store.updateQuantity(...)
  - op === 'clear'   → store.clearCart()
  - op === 'clarify' → store.setQuickReplies(action.options)
  - op === 'upsell'  → show a special chat bubble (handled in chat UI)

  ENDPOINT: POST /chat

  Request body shape:
  {
    messages: Array<{ role: 'user'|'assistant', content: string }>,
    cart: CartItem[],
    profile: UserProfile
  }

  PSEUDOCODE:

  router.post('/', async (req, res) => {
    // 1. Set SSE headers
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-cache');
    res.setHeader('Connection', 'keep-alive');

    // 2. Extract request body
    const { messages, cart, profile } = req.body;

    // 3. Build system prompt (see Section 8)
    const systemPrompt = buildSystemPrompt(cart, profile, menu);

    // 4. Stream from Anthropic SDK
    const stream = await anthropicClient.messages.stream({
      model: 'claude-sonnet-4-20250514',
      max_tokens: 1000,
      system: systemPrompt,
      messages: messages
    });

    // 5. Forward each chunk as SSE
    for await (const chunk of stream) {
      if (chunk.type === 'content_block_delta' &&
          chunk.delta.type === 'text_delta') {
        const text = chunk.delta.text;
        res.write(`data: ${JSON.stringify({ text })}\n\n`);
      }
    }

    // 6. Signal completion
    res.write(`data: ${JSON.stringify({ done: true })}\n\n`);
    res.end();
  });

  export default router;

  FUNCTION: buildSystemPrompt(cart, profile, menu)

  Returns a string. Build it like this (exact template):

  ----------------------------- START OF PROMPT --------------------------------

  You are Bistro, a warm, witty, and helpful AI ordering assistant for The
  Intelligent Bistro restaurant. You have a friendly, slightly playful tone —
  like a knowledgeable waiter who genuinely wants you to have a great meal.

  === MENU ===
  [INSERT JSON.stringify(menu, null, 2) HERE]

  === CURRENT CART ===
  [INSERT JSON.stringify(cart) HERE — if empty, say "The cart is currently empty."]

  === USER DIETARY PROFILE ===
  Restrictions remembered this session: [INSERT profile.restrictions.join(', ') or "None stated"]
  IMPORTANT: If the user orders something that conflicts with their stated
  restrictions, warn them BEFORE adding it. Say "Hey, just so you know, that
  contains [X] — want me to add it anyway?"

  === HOW TO RESPOND ===

  1. Respond conversationally in 1-2 natural sentences BEFORE the action block.
  2. When modifying the cart, embed EXACTLY ONE action block using this format:

     ✦ACTION✦{"op":"...","items":[...]}✦END✦

  3. The action block must be valid JSON. Never put line breaks inside it.
  4. After the action block, you may add 1 more sentence (e.g. a pairing suggestion).
  5. If you need to clarify (user said "chicken sandwich" but there are 2 options),
     use op:"clarify" with a question and options array. Do not add to cart until
     clarified.
  6. After any "add" action, include a natural upsell using op:"upsell" in a
     SECOND action block if there is a relevant pairing. Keep the upsell light
     — "You got the burger — most customers grab the truffle fries with it!"
  7. If the user says something like "I'm vegan" or "no nuts", acknowledge it
     warmly. You do NOT need to emit an action block for dietary preferences —
     the frontend handles storing them separately.
  8. For the recap before checkout (when user says "checkout" or "place order"),
     narrate the entire order naturally before confirming.

  === ACTION SCHEMA ===

  ADD items to cart:
  ✦ACTION✦{"op":"add","items":[{"id":"ITEM_ID","name":"DISPLAY_NAME","qty":NUMBER,"price":PRICE}]}✦END✦

  REMOVE items from cart:
  ✦ACTION✦{"op":"remove","items":[{"id":"ITEM_ID","name":"DISPLAY_NAME","qty":0,"price":0}]}✦END✦

  UPDATE quantity:
  ✦ACTION✦{"op":"update","items":[{"id":"ITEM_ID","name":"DISPLAY_NAME","qty":NEW_QTY,"price":PRICE}]}✦END✦

  CLEAR entire cart:
  ✦ACTION✦{"op":"clear"}✦END✦

  CLARIFY ambiguous order:
  ✦ACTION✦{"op":"clarify","question":"Did you mean the Spicy Chicken or Classic Bistro Burger?","options":["Spicy Chicken Sandwich","Classic Bistro Burger"]}✦END✦

  UPSELL a pairing:
  ✦ACTION✦{"op":"upsell","upsellItem":"truffle-fries","upsellMessage":"Want the truffle fries that most customers pair with it?"}✦END✦

  === RULES ===
  - NEVER invent menu items that are not in the menu above.
  - NEVER output the sentinel markers inside a JSON string value.
  - ALWAYS use the exact item "id" field from the menu, not a guessed ID.
  - Keep responses SHORT. Under 3 sentences total (excluding the action block).
  - The user should feel like they are texting a smart, friendly waiter.

  ----------------------------- END OF PROMPT ----------------------------------

        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ messages, cart, profile })
      })

      reader = response.body.getReader()
      decoder = new TextDecoder()

      loop:
        { done, value } = await reader.read()
        if done: break

        text = decoder.decode(value)

        // SSE format: each event is "data: {...}\n\n"
        // Split by \n\n to get individual events
        events = text.split('\n\n').filter(Boolean)

        for each event:
          if event starts with 'data: ':
            jsonStr = event.slice('data: '.length)
            parsed = JSON.parse(jsonStr)
            if parsed.done:
              onDone()
              return
            if parsed.text:
              onChunk(parsed.text)

  FUNCTION: fetchMenu()
    return fetch(`${API_BASE}/menu`).then(r => r.json())

  import * as SpeechRecognition from 'expo-speech-recognition';

  function useVoiceInput(onTranscript: (text: string) => void) {

    const startListening = async () => {
      const { status } = await SpeechRecognition.requestPermissionsAsync();
      if (status !== 'granted') return;

      SpeechRecognition.start({
        lang: 'en-US',
        interimResults: true,     // show words as they come in
        continuous: false,
        onResult: (result) => {
          setTranscript(result.results[0].transcript);
          if (result.isFinal) {
            onTranscript(result.results[0].transcript);
            stopListening();
          }
        },
        onError: (e) => setError(e.message)
      });

      setIsListening(true);
    };

    const stopListening = async () => {
      await SpeechRecognition.stop();
      setIsListening(false);
    };

    return { isListening, transcript, startListening, stopListening, error };
  }

    - No position change

    - Compute x = baseX + orbitRadius * Math.cos(angle)
    - Compute y = baseY + orbitRadius * Math.sin(angle)

      const angle = useSharedValue(dot.phaseOffset);

      useEffect(() => {
        if (isActive) {
          angle.value = withRepeat(
            withTiming(dot.phaseOffset + Math.PI * 2, {
              duration: dot.orbitSpeed,
              easing: Easing.linear
            }),
            -1,
            false
          );

          // Return to base position
          angle.value = withTiming(dot.phaseOffset, { duration: 600 });
        }
      }, [isActive]);

      const animatedStyle = useAnimatedStyle(() => ({
        transform: [
          { translateX: dot.baseX + dot.orbitRadius * Math.cos(angle.value) },
          { translateY: dot.baseY + dot.orbitRadius * Math.sin(angle.value) }
        ]
      }));

================================================================================
  SECTION 21 — COMMON ERRORS & HOW TO FIX THEM
================================================================================

ERROR: "Cannot read property 'text' of undefined" in stream parser
  FIX: The SSE chunk may contain multiple events separated by \n\n.
       Always split by '\n\n' and process each event individually.

ERROR: Cart updates not triggering animation
  FIX: Make sure isAnimating resets after 600ms. Use setTimeout in the store
       action: setTimeout(() => set({ isAnimating: false }), 600)

ERROR: expo-speech-recognition not working on Android
  FIX: Add RECORD_AUDIO permission to app.json:
       "android": { "permissions": ["RECORD_AUDIO"] }

ERROR: SSE connection drops after 30 seconds
  FIX: Add keep-alive pings. Every 15 seconds, write:
       res.write(':keep-alive\n\n')
       Use setInterval, and clearInterval on res 'close' event.

ERROR: Reanimated "worklet" errors
  FIX: Ensure 'react-native-reanimated/plugin' is the LAST plugin in babel.config.js
       Clear cache: npx expo start --clear

ERROR: Circular orbit not smooth
  FIX: Use Easing.linear for orbit animations. Any easing other than linear
       creates visible "snapping." For the orbit angle animation specifically,
       always use Easing.linear with withRepeat.

ERROR: AI not reliably outputting the sentinel markers
  FIX: Add this to the system prompt:
       "CRITICAL: You MUST always use the exact characters ✦ACTION✦ and ✦END✦.
       These are special Unicode characters (✦ = U+2726). Do not substitute
       them with asterisks, brackets, or any other characters."

ERROR: fetch() streaming not working on Expo Go
  FIX: Use expo-dev-client instead of Expo Go for full native API support.
       Run: npx expo run:android or npx expo run:ios
